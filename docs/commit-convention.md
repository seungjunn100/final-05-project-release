## 커밋 컨벤션

- 작성 시 변경 내용의 핵심과 목적이 명확히 드러나도록 작성
- 하나의 커밋에 하나의 타입만 작성

### 타입

- `feat` : 기능 추가 (새로운 페이지, 버튼, API 연동)
- `fix` : 버그 수정 (동작 오류, 에러 수정)
- `refactor` : 리팩토링 (기능 변화 없이 구조 개선)
- `style` : 스타일 변경 (CSS, Tailwind 클래스, UI 미세조정)
- `docs` : 문서 (README, 컨벤션 문서)
- `chore` : 기타 작업 (설정 파일, 패키지, 빌드 관련)
- `test` : 테스트 (테스트 코드 추가/수정)

### 예제

#### `git commit -m`

- 변경 내용이 명확하고 1가지일 때
- 파일 수가 적고 영향 범위가 좁을 때

```
git commit -m "docs: 커밋 컨벤션 문서 추가"
```

#### `git commit`

- 변경 파일이 많을 때
- 왜 이렇게 바꿨는지 설명이 필요할 때
- 나중에 내가 다시 봐도 헷갈릴 때

```
feat: 설문 결과 기반 추천 페이지 구현

- 설문 점수 요약 로직 추가
- 추천 상품 리스트 UI 구현
- 추천 이유 문구 렌더링
- 결과 페이지에서 구독 버튼 연결
```
